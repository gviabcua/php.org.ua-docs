# strip\_tags

(PHP 4, PHP 5, PHP 7, PHP 8)

strip\_tags — Видаляє теги HTML та PHP з рядка

### Опис

```methodsynopsis
strip_tags(string $string, array|string|null $allowed_tags = null): string
```

Функція намагається повернути рядок, з якого видалені всі NULL-байти, HTML- та PHP-теги із заданого рядка (`string`). Для видалення тегів використовується той самий механізм, що і функції [fgetss()](function.fgetss.md)

### Список параметрів

`string`

Вхідний рядок.

`allowed_tags`

Другий необов'язковий параметр можна використовувати для вказівки тегів, які не потрібно видаляти. Вони вказуються як рядок (string) або масив (array) з PHP 7.4.0. Дивіться приклад нижче щодо формату цього параметра.

> **Зауваження** :
> 
> Коментарі HTML та PHP-теги також будуть видалені. Це жорстко задано в коді і не може бути змінено за допомогою параметра `allowed_tags`

> **Зауваження** :
> 
> Самозакриваються (такі як  
> ) теги XHTML ігноруються і тільки не самозакриваються теги повинні бути використані в `allowed_tags`К Прикладу, для разрешения как`<br>`, так и`<br/>` потрібно зробити таке:
> 
> ```php
> <?php
> strip_tags($input, '<br>');
> ?>
> ```

### Значення, що повертаються

Повертає рядок без тегів.

### список змін

| Версия | Опис |
| --- | --- |
| 8.0.0 | `allowed_tags` тепер допускає значення null. |
| 7.4.0 | `allowed_tags` Тепер альтернативно приймає масив (array). |

### Приклади

**Приклад #1 Приклад використання** strip\_tags()\*\*\*\*

```php
<?php
$text = '<p>Параграф.</p><!-- Комментарий --> <a href="#fragment">Ещё текст</a>';
echo strip_tags($text);
echo "\n";

// Разрешаем <p> и <a>
echo strip_tags($text, '<p><a>');

// Начиная с PHP 7.4.0, строка выше может быть записана как:
// echo strip_tags($text, ['p', 'a']);
?>
```

Результат виконання наведеного прикладу:

```
Параграф. Ещё текст
<p>Параграф.</p> <a href="#fragment">Ещё текст</a>
```

### Примітки

**Увага**

Ця функція не повинна використовуватися для запобігання атакам XSS. Використовуйте більш відповідні функції для цього завдання, такі як [htmlspecialchars()](function.mdspecialchars.md) чи інші механізми, залежно від контексту виведення.

**Увага**

Через те що **strip\_tags()** не перевіряє валідність HTML, то часткові або зламані теги можуть стати видаленням більшої кількості тексту або даних, ніж очікувалося.

**Увага**

Ця функція не змінює атрибути тегів, дозволених за допомогою `allowed_tags`, включаючи такі атрибути як `style`и`onmouseover`, які можуть бути використані пустотливими користувачами при надсиланні тексту, що відображається також і іншим користувачам.

> **Зауваження** :
> 
> Імена тегів у HTML, що перевищують 1023 байти, будуть розглядатися як невалідні незалежно від параметра `allowed_tags`
