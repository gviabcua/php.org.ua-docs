# flush

(PHP 4, PHP 5, PHP 7, PHP 8)

flush - Скидання системного буфера виводу

### Опис

```methodsynopsis
flush(): void
```

Функція очищає системний буфер виведення PHP, при цьому весь вміст буфера відправляється до браузера користувача (з деякими винятками), незалежно від бекенда PHP (CGI, веб-сервер і т.д.).

**flush()** не зможе перевизначити схему буферизації вашого веб-сервера і не впливає на буферизацію браузера на стороні клієнта. Функція також не впливає на механізм буферизації користувача PHP-коду. Це означає, що [проflush()](function.ob-flush.md) повинен бути викликаний перед **flush()**, щоб очистити вихідні буфери, якщо вони використовуються.

Деякі сервери, особливо під керуванням Win32, як і раніше, продовжуватимуть буферизувати виведення вашого скрипта до передачі результатів у браузер.

Серверні модулі для Apache, такі як modgzip, можуть самі виконувати буферизацію, тому **flush()** не призводить до негайної передачі даних клієнту.

Браузер може також буферизувати дані, що до нього надходять перед відображенням. Наприклад, Netscape буферизує текст до тих пір, поки не отримає кінець рядка або початок тега, і не відображатиме таблиці поки не буде помічений тег самої зовнішньої таблиці.

Деякі версії Microsoft Internet Explorer починають відображати сторінку лише після отримання 256 байт виводу, тому може знадобитися надіслати додаткові прогалини перед скиданням, щоб такі браузери відобразили сторінку.

### Список параметрів

Ця функція не має параметрів.

### Значення, що повертаються

Функція не повертає значення після виконання.
