## Функції, визначені користувачем

Наведемо приклад синтаксису, який використовується для опису функцій:

**Приклад #1 Псевдокод демонструє використання функцій**

```php
<?php
function foo($arg_1, $arg_2, /* ..., */ $arg_n)
{
    echo "Пример функции.\n";
    return $retval;
}
?>
```

Всередині функції можна використовувати будь-який коректний PHP-код, у тому числі інші функції та навіть оголошення [класів](language.oop5.basic.html#language.oop5.basic.class)

Імена функцій дотримуються тих самих правил, що й інші мітки в PHP. Коректне ім'я функції починається з букви або символу підкреслення, за яким слідує будь-яка кількість букв, цифр або символів підкреслення. Як регулярний вираз воно може бути виражене так: `^[a-zA-Z_\x80-\xff][a-zA-Z0-9_\x80-\xff]*$`

**Підказка**

Дивіться також [Руководство по именованию](userlandnaming.md)

Функції не повинні бути визначені до їх використання, *виключаючи* той випадок, коли функції визначаються умовно, як показано у двох наступних прикладах.

У випадку, коли функція визначається залежно від будь-якої умови, наприклад, як це показано у двох наведених нижче прикладах, обробка опису функції повинна *передувати* її виклик.

**Приклад #2 Функції, що залежать від умов**

```php
<?php

$makefoo = true;

/* Мы не можем вызвать функцию foo() в этом месте,
   поскольку она ещё не определена, но мы можем
   обратиться к bar() */

bar();

if ($makefoo) {
  function foo()
  {
    echo "Я не существую до тех пор, пока выполнение программы меня не достигнет.\n";
  }
}

/* Теперь мы благополучно можем вызывать foo(),
   поскольку $makefoo была интерпретирована как true */

if ($makefoo) foo();

function bar()
{
  echo "Я существую сразу с начала старта программы.\n";
}

?>
```

**Приклад #3 Вкладені функції**

```php
<?php
function foo()
{
  function bar()
  {
    echo "Я не существую пока не будет вызвана foo().\n";
  }
}

/* Мы пока не можем обратиться к bar(),
   поскольку она ещё не определена. */

foo();

/* Теперь мы можем вызвать функцию bar(),
   обработка foo() сделала её доступной. */

bar();

?>
```

Усі функції та класи PHP мають глобальну область видимості - вони можуть бути викликані поза функцією, навіть якщо були визначені всередині та навпаки.

PHP не підтримує перевантаження функції, також немає можливості перевизначити або видалити оголошену раніше функцію.

> **Зауваження**: Імена функцій реєстронезалежні для символів ASCII від `A` до `Z`Тим не менш, краще викликати функції так, як вони були оголошені.

Функції PHP підтримують як [списки аргументів змінної довжини](functions.arguments.html#functions.variable-arg-list), так і [значения аргументов по умолчанию](functions.arguments.html#functions.arguments.default). Дивіться також опис функцій [funcnumargs()](function.func-num-args.html) [funcgetarg()](function.func-get-arg.html) і [funcgetargs()](function.func-get-args.html) для детальнішої інформації.

Можна викликати функції PHP рекурсивно.

**Приклад #4 Рекурсивні функції**

```php
<?php
function recursion($a)
{
    if ($a < 20) {
        echo "$a\n";
        recursion($a + 1);
    }
}
?>
```

> **Зауваження**: Рекурсивний виклик методів/процедур з глибиною понад 100-200 рівнів рекурсії може спричинити переповнення стека та призвести до аварійного завершення скрипту. Зокрема, нескінченна рекурсія вважатиметься програмною помилкою.
