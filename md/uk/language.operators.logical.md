## Логічні оператори

**Логічні оператори**

| Пример | Название | Результат |
| --- | --- | --- |
| $a and $b | І | \*\*`true`\*\*якщо і $a, і $b **`true`** |
| $a або $b | Або | **`true`**, якщо або $a, або $b **`true`** |
| $a xor $b | Виключне або | \*\*`true`\*\*якщо $a, або $b \*\*`true`\*\*але не обидва. |
| ! $a | Заперечення | \*\*`true`\*\*якщо $a не **`true`** |
| $a && $b | І | \*\*`true`\*\*якщо і $a, і $b **`true`** |
| $а |  | $в |

Сенс двох різних варіантів для операторів "and" та "or" у тому, що вони працюють з різними пріоритетами (див. таблицю [Пріоритет виконання операторів](language.operators.precedence.md)

**Приклад #1 Пояснення логічних операторів**

```php
<?php

// --------------------
// foo() никогда не будет вызвана, т.к. эти операторы являются шунтирующими (short-circuit)

$a = (false && foo());
$b = (true  || foo());
$c = (false and foo());
$d = (true  or  foo());

// --------------------
// "||" имеет больший приоритет, чем "or"

// Результат выражения (false || true) присваивается переменной $e
// Действует как: ($e = (false || true))
$e = false || true;

// Константа false присваивается $f, а затем значение true игнорируется
// Действует как: (($f = false) or true)
$f = false or true;

var_dump($e, $f);

// --------------------
// "&&" имеет больший приоритет, чем "and"

// Результат выражения (true && false) присваивается переменной $g
// Действует как: ($g = (true && false))
$g = true && false;

// Константа true присваивается $h, а затем значение false игнорируется
// Действует как: (($h = true) and false)
$h = true and false;

var_dump($g, $h);
?>
```

Результатом виконання цього прикладу буде щось подібне:

```
bool(true)
bool(false)
bool(false)
bool(true)
```
