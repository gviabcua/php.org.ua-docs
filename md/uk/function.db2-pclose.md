# db2pclose

(PECL ibmdb2> = 1.8.0)

db2pclose — Закриває постійне з'єднання з базою даних

### Опис

```methodsynopsis
db2_pclose(resource $resource): bool
```

Функція закриває клієнтське з'єднання DB2, створене за допомогою [db2pconnect()](function.db2-pconnect.html) та повертає відповідні ресурси серверу бази даних.

> **Зауваження**
> 
> Ця функція доступна лише для i5/OS у відповідь на запити системного адміністратора i5/OS.

Якщо у вас є постійне з'єднання DB2, створене за допомогою [db2pconnect()](function.db2-pconnect.html), можна використовувати цю функцію, щоб закрити з'єднання. Щоб уникнути значних втрат продуктивності з'єднання, цю функцію слід використовувати лише в окремих випадках, коли постійне з'єднання перестало відповідати або постійне з'єднання не буде потрібно протягом тривалого часу.

### Список параметрів

`connection`

Встановлює активне з'єднання клієнта DB2.

### Значення, що повертаються

Повертає **`true`** у разі успішного виконання або **`false`** у разі виникнення помилки.

### Приклади

**Приклад #1 Закриття постійного з'єднання**

У цьому прикладі демонструється успішна спроба закрити з'єднання з базою даних IBM DB2 i5/OS.

```php
<?php
$conn = db2_pconnect('', '', '');
$rc = db2_pclose($conn);
if ($rc) {
    echo "Соединение было успешно закрыто.";
}
?>
```

Результат виконання цього прикладу:

```
Соединение было успешно закрыто.
```
