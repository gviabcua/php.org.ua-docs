## Використання просторів імен: перехід до глобальної функції/константи

(PHP 5> = 5.3.0, PHP 7, PHP 8)

Усередині простору імен, коли PHP зустрічає неповне ім'я класу, функції або константи, він перетворює ці імена з різними пріоритетами. Імена класів завжди перетворюються на поточне ім'я простору імен. Таким чином, щоб отримати доступ до внутрішнього класу або класу користувача поза простір імен, необхідно посилатися за їх абсолютним ім'ям. Наприклад:

**Приклад #1 Доступ до глобальних класів усередині простору імен**

```php
<?php
namespace A\B\C;
class Exception extends \Exception {}

$a = new Exception('hi'); // $a - это объект класса A\B\C\Exception
$b = new \Exception('hi'); // $b - это объект класса Exception

$c = new ArrayObject; // фатальная ошибка, класс A\B\C\ArrayObject не найден
?>
```

Для функцій і констант, PHP буде вдаватися до глобальних функцій чи константів, якщо функція чи константа немає у просторі імен.

**Приклад #2 Необхідність вдатися до глобальних функцій/константів усередині простору імен**

```php
<?php
namespace A\B\C;

const E_ERROR = 45;
function strlen($str)
{
    return \strlen($str) - 1;
}

echo E_ERROR, "\n"; // выводит "45"
echo INI_ALL, "\n"; // выводит "7" - прибегнет к глобальной INI_ALL

echo strlen('hi'), "\n"; // выводит "1"
if (is_array('hi')) { // выводит строку "это не Масив"
    echo "это Масив\n";
} else {
    echo "это не Масив\n";
}
?>
```
