## Простори імен: повернення до глобального простору для функцій та констант

(PHP 5 >= 5.3.0, PHP 7, PHP 8)

Коли всередині простору ім'я PHP зустрічає неповне ім'я класу, функції або контекст константи, він дозволяє ці імена з різними пріоритетами. Імена класів дозволяються в ім'я простору імен. Тому, щоб отримати доступ до внутрішнього класу або класу користувача поза простором імен, необхідно звертатися до них по абсолютному імені. Наприклад:

**Приклад #1 Доступ до глобальних класів у просторі імен**

```php
<?php

namespace A\B\C;
class Exception extends \Exception {}

$a = new Exception('hi'); // $a — это объект класса A\B\C\Exception
$b = new \Exception('hi'); // $b — это объект класса Exception

$c = new ArrayObject; // фатальная ошибка, класс A\B\C\ArrayObject не найден

?>
```

PHP буде звертатися до глобальних функцій або константів, якщо функція або константа не існує в поточному просторі імен.

**Приклад #2 Повернення до глобальних функцій чи константів усередині простору імен**

```php
<?php

namespace A\B\C;

const E_ERROR = 45;
function strlen($str)
{
    return \strlen($str) - 1;
}

echo E_ERROR, "\n"; // выводит 45
echo INI_ALL, "\n"; // выводит «7» — возвращается к глобальной константе INI_ALL

echo strlen('hi'), "\n"; // выводит «1»
if (is_array('hi')) { // выводит строку «это не массив»
    echo "это массив\n";
} else {
    echo "это не массив\n";
}

?>
```
