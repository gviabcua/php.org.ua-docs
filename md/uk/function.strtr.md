# strtr

(PHP 4, PHP 5, PHP 7, PHP 8)

strtr — Перетворення заданих символів або заміна підрядків

### Опис

```methodsynopsis
strtr(string $string, string $from, string $to): string
```

Альтернативна сигнатура (не підтримується з іменованими аргументами):

```methodsynopsis
strtr(string $string, array $replace_pairs): string
```

Якщо вказано три аргументи, ця функція повертає копію `string`, в якій всі входження кожного символу (однобайтного) з `from` були замінені на відповідний символ у параметрі `to`, тобто кожне входження з `$from[$n]` замінюється на `$to[$n]`, де `$n` - це коректне усунення рядка в обох аргументах.

Якщо довжина рядків `from` і `to` відрізняються, "зайві" символи у довшому рядку не використовуються. Довжина значення, що повертається, буде такою ж, як і довжина `string`

**strtr()** може викликатися із двома аргументами. В цьому випадку `from` має бути масивом (array), у формі `array('from' => 'to', ...)`. Функція поверне рядок, у якому всі ключі масиву замінять їх елементами . **strtr()** в першу чергу замінює довші ключі, причому один і той же рядок пошуку використовується лише один раз.

У цьому випадку ключі та значення можуть мати будь-яку довжину, за винятком того, що ключі не повинні бути порожніми. Крім того, довжина значення, що повертається, може відрізнятися від `string`. Однак, варто врахувати той факт, що ця функція є максимально ефективною у тому випадку, якщо всі ключі мають однаковий розмір.

### Список параметрів

`string`

Замінний рядок (string).

`from`

Рядок (string), який буде замінено на рядок `to`

`to`

Рядок (string), що замінює рядок `from`

`replace_pairs`

Параметр `replace_pairs` може бути використаний замість `to` і `from`, у разі він є масивом (array) і має форму `array('from' => 'to', ...)`

Якщо `replace_pairs` містить ключ, який є порожнім рядком (string) (`""`), елемент ігнорується; починаючи з PHP 8.0.0 у цьому випадку викликається **`E_WARNING`**

### Значення, що повертаються

Повертає рядок (string) із проведеними у ньому замінами.

### Приклади

**Приклад #1 Приклад використання **strtr()****

```php
<?php
//В этой форме strtr() осуществляет побайтное преобразование
//Таким образом, здесь подразумевается однобайтная кодировка:
$addr = strtr($addr, "äåö", "aao");
?>
```

Наступний приклад демонструє поведінку функції **strtr()** при виклику лише з двома аргументами. Зверніть увагу на пріоритет замін (`"h"` не використовується, тому що існують довші збіги) і як замінений текст більше не бере участі в пошуку.

**Приклад #2 Приклад використання **strtr()** з двома аргументами**

```php
<?php
$trans = array("h" => "-", "hello" => "hi", "hi" => "hello");
echo strtr("hi all, I said hello", $trans);
?>
```

Результат виконання цього прикладу:

```
hello all, I said hi
```

Ці два режими досить сильно відрізняються. При виклику **strtr()** з трьома аргументами замінюються байти, а при виклику з двома аргументами можуть бути замінені і довші підрядки.

**Приклад #3 Порівняння поведінки функції **strtr()****

```php
<?php
echo strtr("baab", "ab", "01"),"\n";

$trans = array("ab" => "01");
echo strtr("baab", $trans);
?>
```

Результат виконання цього прикладу:

```
1001
ba01
```
