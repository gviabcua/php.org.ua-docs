# svndelete

(PECL svn >= 0.4.0)

svndelete — Видалення елементів із робочої копії або репозиторію.

### Опис

```methodsynopsis
svn_delete(string $path, bool $force = false): bool
```

Видаляє файл, каталог або символічне посилання `path` з робочої копії. Елемент буде видалено з репозиторію під час наступного виклику [svncommit()](function.svn-commit.html) з робочої копії.

### Список параметрів

`path`

Шлях до елемента, що видаляється.

> **Зауваження**: Відносні шляхи будуть обчислені, якби поточна робоча директорія була домашньою папкою самого PHP Щоб використовувати робочу директорію скрипта, що викликає, використовуйте [realpath()](function.realpath.md) або dirname(FILE

`force`

При **`true`** файл буде видалено, навіть якщо в ньому є локальні зміни. При **`false`** локальні модифікації спричинять збій під час виконання операції. За замовчуванням **`false`**

### Значення, що повертаються

Повертає **`true`** у разі успішного виконання або **`false`** у разі виникнення помилки.

### Примітки

**Увага**

Ця функція є *ЕКСПЕРИМЕНТАЛЬНОЇ*. Поведінка цієї функції, її ім'я та документація, що до неї належить, можуть змінитися в наступних версіях PHP без повідомлення. Використовуйте цю функцію на свій страх та ризик.
