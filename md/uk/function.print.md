# print

(PHP 4, PHP 5, PHP 7, PHP 8)

print — Виводить рядок

### Опис

```methodsynopsis
print(string $expression): int
```

Виводить вираз `expression`

`print` - це не функція, а мовна конструкція. Її аргумент - це вираз після ключового слова `print`, не обмежена круглими дужками.

Основні відмінності від конструкції [echo](function.echo.md) полягають у тому, що конструкція `print` приймає лише один аргумент і завжди повертає

### Список параметрів

`expression`

Вираз для виведення. Нестрокові значення будуть перетворені на рядки, навіть якщо [включена директива`strict_types`](language.types.declarations.md#language.types.declarations.strict)

### Значення, що повертаються

Завжди повертає

### Приклади

**Приклад #1 Приклад використання мовної конструкції `print`**

```php
<?php

print "print не требует скобок.";

// Новая строка или пробел не добавляются; ниже выводит «приветмир» в одну строку
print "привет";
print "мир";

print "Эта строка занимает
несколько строк. Новые строки
также будут выведены";

print "Эта строка занимает\nнесколько строк. Новые строки\nтакже будут выведены";

// Аргументом может быть любое выражение, производящее строку
$foo = "Приклад";
print "foo — это $foo"; // foo — это Приклад

$fruits = ["лимон", "апельсин", "банан"];
print implode(" и ", $fruits); // лимон и апельсин и банан

// Нестроковые выражения приводятся к строковым, даже если установлено declare(strict_types=1)
print 6 * 7; // 42

// Поскольку у конструкции print есть возвращаемое значение, её разрешено вызвать в выражениях
// Следующие выходные данные «привет мир»
if ( print "привет" ) {
    echo " мир";
}

// следующее выражение выведет «true»
( 1 === 1 ) ? print 'true' : print 'false';
?>
```

### Примітки

> **Зауваження** **Використання з круглими дужками**
> 
> Заключение аргумента`print` у круглі дужки не викликає синтаксичної помилки та створить синтаксис, який виглядає як звичайний виклик функції. Однак це може ввести в оману, оскільки круглі дужки — частиною виразу, а не частина самого синтаксису. `print`
> 
> ```php
> <?php
> 
> print "привіт";
> // виведе «привіт»
> 
> print("привіт");
> // також виведе "привіт", тому що ("привіт") - коректне вираження
> 
> print(1 + 2) * 3;
> // Виведе «9»; круглі дужки призводять до того, що спочатку обчислюється 1+2, а потім 3*3
> // конструкція print бачить весь вираз як один аргумент
> 
> if ( print("привіт") && false ) {
>     print "- усередині if";
> }
> else {
>     print "- всередині else";
> }
> // виведе «- всередині if»
> // спочатку обчислюється вираз («привіт») && false, даючи false
> // це наводиться до порожнього рядка " " і виводиться
> // конструкція print потім повертає 1 тому виконується код в блоці if
> 
> ?>
> ```
> 
> При виклику `print` у більшому вираженні може знадобитися розміщення ключового слова та його аргументу в круглих дужках для отримання потрібного результату:
> 
> ```php
> <?php
> 
> if ( (print "привіт") && false ) {
>     print "- усередині if";
> }
> else {
>     print "- всередині else";
> }
> // виведе «привіт - усередині else»
> // на відміну попереднього прикладу, вираз (print " привіт " ) обчислюється першим
> // після виведення «привіт» print повертає 1
> // оскільки 1 && false - хибно, виконується код у блоці else
> 
> print "привіт" && print "світ";
> // виведе «Мір1»; print "світ" виконується в першу чергу,
> // тоді вираз "привіт" && 1 передається в ліву частину print
> 
> (print "привіт") && (print "світ");
> // Виведе «привіт світ»; круглі дужки змушують вирази print
> // Виконується перед оператором &&
> ?>
> ```

> **Зауваження**: Оскільки це мовна конструкція, а не функція, її не можна викликати як [змінну функцію](functions.variable-functions.md) або передавати як [іменований аргумент](functions.arguments.md#functions.named-arguments)
