# Налаштування під час виконання

Поведінка цих функцій залежить від установок у php.ini.

**Опції налаштування mongodb**

| Имя | По умолчанию | Место изменения | Список изменений |
| --- | --- | --- | --- |
| [mongodb.debug](mongodb.configuration.md#ini.mongodb.debug) | "" | **`INI_ALL`** |  |

Коротке пояснення конфігураційних директив.

`mongodb.debug`string

Цей параметр можна використовувати для увімкнення або вимкнення ведення журналу налагодження на рівні трасування в драйвері (і libmongoc).

Укажите пустую строку,`"0"` `"off"` `"no"`или`"false"`, щоб вимкнути ведення журналу.

Укажите`"stderr"`или`"stdout"`, щоб вести журнал у `stderr`или`stdout`відповідно.

Укажите`"1"` `"on"` `"yes"`или`"true"` для ведення журналу новий тимчасовий файл в системному тимчасовому каталозі за промовчанням (тобто . [sys\_get\_temp\_dir()](function.sys-get-temp-dir.md)

Вкажіть будь-який інший рядок для ведення журналу до нового тимчасового файлу в цьому каталозі. Якщо каталог не може бути використаний, замість нього використовуватиметься системний часовий каталог за замовчуванням.

> **Зауваження** :
> 
> Зверніть увагу, що журнал налагодження може містити конфіденційну інформацію, таку як облікові дані на сервері MongoDB та повні документи, записані чи прочитані із сервера. Будь ласка, перегляньте всі журнали налагодження, перш ніж ділитися ними з іншими людьми.
