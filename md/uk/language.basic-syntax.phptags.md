## Мітки PHP

Коли PHP обробляє файл, він шукає теги, що відкривають і закривають, такі як `<?php` і `?>`, які вказують PHP, коли починати та закінчувати обробку коду між ними. Подібний спосіб обробки дозволяє PHP впроваджуватися у всі види різних документів, так як все, що знаходиться поза парою тегів, що відкривають і закривають, буде проігноровано парсером PHP.

PHP включає короткий echo-тег `<?=`, який є скороченням для більш багатослівного `<?php echo`

**Приклад #1 Теги, що відкривають і закривають PHP**

```php
1.  <?php echo 'если вы хотите хранить код PHP в документах XHTML или XML,
                то используйте эти теги'; ?>

2. Вы можете использовать короткий 'echo'-тег чтобы <?= 'напечатать эту строку' ?>.
    Этот тег эквивалентен такому коду
    <?php echo 'напечатать эту строку' ?>.

3.  <? echo 'этот код с короткими тегами, но он будет работать только если '.
                'включена опция "short_open_tag"'; ?>
```

Короткі теги (третій приклад) доступні за замовчуванням, але їх можна вимкнути за допомогою директиви [shortopentag](ini.core.md#ini.short-open-tag) у конфігураційному файлі php.ini або відключені за замовчуванням, якщо PHP був скомпільований з опцією **\-disable-short-tags**

> **Зауваження**
> 
> Оскільки короткі теги можна вимкнути, рекомендується використовувати лише звичайні теги (`<?php ?>` і `<?= ?>`) для максимальної сумісності.

Якщо файл містить лише код PHP, бажано опустити тег, що закриває, в кінці файлу. Це допомагає уникнути додавання випадкових символів пропуску або перекладу рядка після закриття тега PHP, які можуть спричинити небажані ефекти, оскільки PHP починає виводити дані в буфер за відсутності наміру у програміста виводити будь-які дані в цій точці скрипта.

```php
<?php
echo "Hello world";

// ... ещё код

echo "Последнее выражение";

// Скрипт заканчивается тут без закрывающего тега PHP
```
